## 컨벤션 📋

### 💭 Commit
```
# style      코드의 구조/형태 개선
# remove      코드/파일 삭제
# bugfix      버그 수정
# hotfix      긴급 수정
# feat      새 기능
# docs      문서 추가/수정
# init      프로젝트 시작
# test      테스트 추가/수정
# release      릴리즈/버전 태그
# chore      구성 파일 추가/삭제
# refactor      코드 리팩토링
# fix      간단한 수정
```

### 💭 Branch
```
main : 실제 프로덕트 브랜치
develop : 신규 버전 개발 브랜치
feat : 기능 구현 브랜치
fix : 기능 수정 브랜치
refactor : 리팩토링 브랜치
bugfix : 버그 수정 브랜치
```
